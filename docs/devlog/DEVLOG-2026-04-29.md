### Key design decisions
- **Unstructured client for HelmRelease:** Rather than importing `helm.toolkit.fluxcd.io` API types from the Flux SDK (which would add a heavy transitive dependency and lock the operator to a specific Flux version), `ClusterScanner` uses `k8s.io/apimachinery/pkg/apis/meta/v1/unstructured` to list HelmRelease objects by GVK and extract fields via `NestedString`. If the CRD is absent the lister returns `apimeta.IsNoMatchError` which is treated as "no releases found" â€” making the operator safe to run in clusters without Flux.
- **CompatibilityPolicy triggers StackUpgrade creation:** On each reconcile loop the controller fetches installed tools from the scanner, calls `Matrix.LatestVersion` for each, and if the installed version differs it calls `Matrix.Resolve` to confirm the upgrade path exists. It then checks whether a StackUpgrade named `auto-{toolname}-{version}` already exists in the policy's namespace; if not, it creates one with `approvalRequired: true`. This prevents duplicate CRs on repeated reconcile loops without needing ownerReferences or finalizers.
- **Naming pattern for auto-created upgrades:** Auto-created StackUpgrade names follow the pattern `auto-{toolname}-{targetVersion}` (e.g. `auto-cert-manager-1.15`). The fixed name doubles as an idempotency key â€” a `Get` before `Create` short-circuits if the object exists, so repeated scans never produce duplicates even if the upgrade is sitting in Pending state.
