# Kube-Viltrumite — development log

## Project overview
Kube-Viltrumite is a Kubernetes operator for AI-powered upgrade planning, named after the Viltrumites from Invincible — superior beings who impose order. It is built for platform engineers and SREs who manage clusters running multiple Helm-deployed tools (cert-manager, ingress-nginx, external-secrets, etc.) and need structured, risk-aware upgrade plans. The operator watches for installed Helm releases via FluxCD's HelmRelease CRD, cross-references a YAML knowledge base of version compatibility data, and uses a pluggable AI provider interface to analyze changelogs and produce human-readable upgrade guidance — all expressed as Kubernetes custom resources.

## Architecture summary
Kube-Viltrumite follows the standard kubebuilder operator pattern with two CRDs: `StackUpgrade` (a single upgrade execution request) and `CompatibilityPolicy` (a cluster-wide policy that continuously scans for outdated tools and auto-creates StackUpgrade objects). The `CompatibilityPolicyReconciler` calls the `ClusterScanner` to list FluxCD `HelmRelease` objects using the unstructured client — avoiding a hard import of the Flux SDK — then queries the `Matrix` resolver against an embedded YAML knowledge base to determine if upgrades are available. When an upgrade is warranted it creates a `StackUpgrade` CR. The `StackUpgradeReconciler` drives the upgrade through a phase state machine: Pending → Approved → Upgrading → Succeeded (or Failed → RolledBack). All AI analysis is done through the `AIProvider` interface, keeping the operator decoupled from any specific backend; adapters exist for Anthropic, OpenAI-compatible APIs, Ollama, and a no-op fallback.

## Known issues and workarounds
- **go version mismatch:** Local toolchain reports go1.22.5 but go.mod targets 1.23. kubebuilder refused to init. Workaround: `--skip-go-version-check` flag; building with go1.23+ will be required in CI.
- **kubebuilder init blocked by existing files:** go.mod, Makefile, and cmd/ already existed from the manual scaffold. kubebuilder would have overwritten them. Workaround: manually recreated the kubebuilder-expected artifacts (PROJECT, config/, groupversion_info.go, zz_generated.deepcopy.go, tools/tools.go) without running kubebuilder init.
- **main.go does not wire controllers:** The manager starts cleanly but neither reconciler is registered. The operator is not functional until Session 10 is complete.
