## Key Decisions

1. **`Upgrader` interface in controller package** — the controller defines the interface it needs; the executor package satisfies it. This is standard Go interface ownership (consumer owns the interface) and matches the CLAUDE.md rule that the operator never imports concrete types directly.

2. **`planStep.Risk` omitted** — task spec asked to set `Risk: step.Risk`, but `planner.UpgradeStep` has no `Risk` field. Setting it would have been a compile error. Correct to omit.

3. **`RequeueAfter: 5s` on success** — matches CLAUDE.md convention (`RequeueAfter` not `Requeue: true`) and the `reconcileApproved` pattern already in the file.

4. **Inner `statusErr` variable on failure paths** — the error from `r.Status().Update` is returned (not the upgrade error), so the reconciler retries correctly. The original upgrade/result error is captured in `FailureReason` on the status object.
