## Key Decisions

1. **`newActionConfig` factory field** — added as an unexported field beyond the spec's three struct fields. Required for test isolation without a real cluster; the factory is replaced in tests with a memory-backed config. This is the standard Go pattern for injectable dependencies on concrete types.

2. **Memory driver + `chartutil.DefaultCapabilities`** — `action.Configuration.Capabilities` must be pre-set in tests; otherwise `getCapabilities()` calls the Kubernetes discovery API. Setting it to `chartutil.DefaultCapabilities` avoids any cluster call.

3. **`FailingKubeClient{BuildError}` for Test 4** — Helm's `validateManifest` calls `KubeClient.Build` inside `prepareUpgrade`, which runs before any dry-run short-circuit. A `BuildError` reliably forces `upgrade.Run` to return an error in dry-run mode without needing a real chart template render failure.

4. **Pre-flight errors vs upgrade errors** — pre-flight failures (release absent, chart not found) return `(nil, error)`. Only failures from `upgrade.Run` return `(*UpgradeResult{RolledBack: true}, nil)`. This split lets callers distinguish "couldn't start" from "started and failed."

## Code Review Findings

**MAJOR-1 — Context not propagated to upgrade (line 112)**
`upgrade.Run(...)` discards the caller's `ctx` (creates `context.Background()` internally). Should call `upgrade.RunWithContext(ctx, step.ReleaseName, ch, currentValues)`. Impact: operator reconciler cancellations and deadlines are not observed during active upgrades.

**MAJOR-2 — All `action.NewGet` errors mis-labelled "release not found" (lines 77–79)**
Cluster connectivity failures, RBAC errors, and TLS errors are all wrapped with `"release not found: ..."`. Should `errors.Is(err, driver.ErrReleaseNotFound)` and use a separate message for other errors.

**MAJOR-3 — `RolledBack: true` set even when no rollback was attempted (lines 117–125)**
`Atomic=true` rollback only fires when `performUpgrade` fails after writing a release revision. Failures in `prepareUpgrade` (our Test 4 scenario) never touch storage, so `RolledBack: true` misrepresents the state. Mitigation: inspect error for Helm's `"has been rolled back due to atomic being set"` marker.
